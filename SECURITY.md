# Política de Segurança

## Reportando vulnerabilidades

**Não abra issue público para vulnerabilidades.** Use:

1. GitHub Security Advisories (preferencial — "Report a vulnerability").
2. Email: `security@<seu-domínio>` (PGP opcional).

Inclua:
- Descrição do problema
- Vetor / passos para reproduzir (controlado)
- Severidade estimada
- Versão / commit afetado

## SLA de resposta

| Severidade | Triagem | Mitigação | Patch |
|------------|---------|-----------|-------|
| Crítica    | 24h     | 72h       | 7d    |
| Alta       | 72h     | 14d       | 30d   |
| Média      | 7d      | 30d       | 60d   |
| Baixa      | 14d     | 60d       | 90d   |

## Escopo

Cobre o código deste repositório e dependências diretas. Issues de
infraestrutura de terceiros devem ser reportadas ao fornecedor.

## Hall of fame

Reconhecemos pesquisadores que reportaram via canal privado e seguiram
divulgação responsável.
